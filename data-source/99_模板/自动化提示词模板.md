# 自动化提示词模板

> 建立日期：2026-06-21
> 定位：Codex 自动化任务提示词模板。

## 日频候选信息收集

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：daily。读取 README.md、metadata/automation-plan.json、metadata/taxonomy.json、metadata/sources.json、metadata/direction-registry.json、metadata/source-registry.json、metadata/update-state.json 和 data-source/05_信息候选池/。优先按 metadata/update-state.json 的 period_policy.active_research_period 选择或新建 data-source/05_信息候选池/YYYYQn_信息候选池.md；未配置时再按当前日期选择。如果新建周期文件，必须同步 frontmatter、更新记录和 metadata/sources.json。优先使用 metadata/source-registry.json 中登记的信息源，按 metadata/direction-registry.json 中的方向总表识别已有方向；如果发现未登记但满足入库标准的新方向，写入候选池并标记 new_direction_candidate。日频任务只追加候选池，不直接修改正式方向研究、历史案例、观察清单或复盘记录。每条候选信息必须包含日期、来源层级、来源、方向、信息摘要、初步判断、处理结果。如果网络不可用或来源无法访问，不创建空候选信息，不更新正式文档；更新 metadata/update-state.json 的 daily.last_status 为 network_limited，daily.last_error 写明原因，运行本地校验并报告。正常完成时更新 metadata/update-state.json 的 daily 状态，运行 node scripts/validate-knowledge-base.js，最后用中文报告新增候选数量、涉及方向、新方向候选、review 项和校验结果。
```

## 周频候选池复核

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：weekly。读取 README.md、metadata/automation-plan.json、metadata/taxonomy.json、metadata/sources.json、metadata/direction-registry.json、metadata/source-registry.json、metadata/update-state.json 和本窗口内 data-source/05_信息候选池/ 的候选信息。优先按 metadata/update-state.json 的 period_policy.active_research_period 选择或新建 data-source/03_观察清单/YYYYQn_观察清单.md；未配置时再按当前日期选择。按 direction-registry 中的方向状态决定处理优先级；对 new_direction_candidate 判断是否进入 candidate_directions、升级为 watching，或新建正式方向研究并升级为 tracking。对已有方向，按趋势、瓶颈、资本开支、钱流向谁、稀缺性、市场定价、验证信号判断是否更新正式文档。处理后必须回写候选记录处理结果和处理统计，避免已处理信息在下一窗口重复处理。网络不可用时继续处理本地候选池和已有文档；需要外部来源确认的事项标记 review，不得编造来源。同步候选池、frontmatter、文档内更新记录、metadata/sources.json、metadata/direction-registry.json、CHANGELOG.md 和 metadata/update-state.json。运行 node scripts/validate-knowledge-base.js，最后用中文报告本周候选数量、正式更新文件、方向状态变化、新方向候选、review 项、网络状态和校验结果。
```

## 月频观察清单更新

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：monthly。读取 README.md、metadata/automation-plan.json、metadata/taxonomy.json、metadata/sources.json、metadata/direction-registry.json、metadata/source-registry.json、metadata/update-state.json、候选池、方向研究和当前观察清单。优先按 metadata/update-state.json 的 period_policy.active_research_period 选择或新建 data-source/03_观察清单/YYYYQn_观察清单.md；未配置时再按当前日期选择。基于本月候选信息、方向状态和已更新正式文档，更新观察方向、要验证的问题、跟踪信号和优先级。priority 和 tracking 方向优先进入观察清单；watching 方向只在证据增强时进入；paused、archived、rejected 默认不主动更新。网络不可用时继续基于本地候选池、方向研究和观察清单更新；需要外部确认的事项标记 review。同步更新记录、CHANGELOG.md、metadata/sources.json 和 metadata/update-state.json。运行 node scripts/validate-knowledge-base.js，最后报告本月调整了哪些观察方向、依据是什么、哪些方向状态发生变化、哪些事项仍为 review、网络状态和校验结果。
```

## 季度判断复盘

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：quarterly。读取 README.md、metadata/automation-plan.json、metadata/taxonomy.json、metadata/sources.json、metadata/direction-registry.json、metadata/source-registry.json、metadata/update-state.json、候选池、方向研究、观察清单和复盘记录。按当前年份选择或新建 data-source/04_复盘记录/YYYY_判断复盘.md，并按 1/4/7/10 月 1 日规则复盘上一自然季度。基于本季度已记录的信息，更新复盘记录：总结哪些方向判断被验证、哪些被削弱、哪些应暂停或归档、哪些新方向应升级。必要时同步 metadata/direction-registry.json 的方向状态；复盘完成后，将 active_research_period 切换或确认到运行日所在自然季度，并选择或新建对应 data-source/03_观察清单/YYYYQn_观察清单.md，调整该季度观察重点。网络不可用时继续基于本地记录完成复盘；无法外部验证的结论标记 review。不要生成投资建议，只记录判断链条变化和证据。同步更新记录、CHANGELOG.md、metadata/sources.json 和 metadata/update-state.json。运行 node scripts/validate-knowledge-base.js，最后报告本季度复盘结论、方向状态变化、当前季度观察重点、更新文件、review 项、网络状态和校验结果。
```

## 重大事件处理

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：event。处理以下重大事件信息：[在这里粘贴事件、链接或文件路径]。读取 metadata/direction-registry.json 和 metadata/source-registry.json，判断该事件属于已有方向、新方向候选还是无关信息。先写入候选池；如来源和证据足够，自动更新相关正式文档、方向状态、metadata/sources.json、metadata/direction-registry.json、CHANGELOG.md 和 metadata/update-state.json。遇到证据冲突、来源不足或方法论重大变化时标记 review。运行 node scripts/validate-knowledge-base.js，最后报告处理结果、影响方向、是否更新正式文档和校验结果。
```
