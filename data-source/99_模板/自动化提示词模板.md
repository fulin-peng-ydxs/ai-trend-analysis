# 自动化提示词模板

> 建立日期：2026-06-21
> 定位：Codex 自动化任务提示词模板。

## 日频候选信息收集

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：daily。执行日频候选信息收集：只追加信息候选池，不直接修改正式方向研究、历史案例、观察清单、复盘记录、公司注册表或核心标的观察池。已有方向或已有公司相关信息标记为 watch；满足入库标准但不属于已有方向的信息标记为 new_direction_candidate；不得在日频中使用 update 或 create。最后报告新增候选数量、涉及方向、新方向候选、review 项和校验结果。
```

## 周频候选池复核

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：weekly。执行周频候选池复核：处理本窗口候选信息，回写候选记录处理结果和处理统计，避免下个窗口重复处理。对 new_direction_candidate 判断是否登记、升级或新建正式方向；对已有方向按趋势、瓶颈、资本开支、钱流向谁、稀缺性、市场定价和验证信号判断是否更新正式文档。公司相关信息按公司注册表策略判断是否更新公司注册表、核心标的观察池或公司画像。只有实际写入正式文档时才把候选记录标记为 update 或 create；未写入但仍需观察的保持 watch；需要人工判断的标记 review。最后报告本周候选数量、正式更新文件、方向或公司状态变化、新方向候选、review 项和校验结果。
```

## 月频观察清单更新

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：monthly。执行月频观察清单更新：基于本月候选信息、方向状态、公司注册表和已更新正式文档，更新观察方向、待验证问题、跟踪信号和优先级。priority 和 tracking 方向优先进入观察清单；watching 方向只在证据增强时进入；paused、archived、rejected 默认不主动更新。同步更新公司层验证信号和需要复盘的问题。最后报告本月调整了哪些观察方向和公司层信号、依据是什么、哪些事项仍为 review、校验结果。
```

## 季度判断复盘

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：quarterly。执行季度判断复盘：复盘上一自然季度，更新年度复盘记录，判断哪些方向或公司层判断被验证、削弱、证伪、暂停、归档或升级。复盘完成后，确认运行日所在季度的 active_research_period，并调整该季度观察重点。不得生成投资建议，只记录判断链条变化、证据和后续验证问题。最后报告本季度复盘结论、方向或公司状态变化、当前季度观察重点、更新文件、review 项和校验结果。
```

## 重大事件处理

```text
Use $ai-trend-knowledge-maintainer 维护 /Users/pengshuaifeng/ai-trend-analysis。任务类型：event。处理以下重大事件信息：[在这里粘贴事件、链接或文件路径]。先写入候选池，再判断是否影响已有方向、新方向候选、公司注册表、核心标的观察池、公司画像或复盘记录。来源和证据足够时更新相关正式文档；证据冲突、来源不足或方法论重大变化时标记 review。最后报告处理结果、影响方向或公司、是否更新正式文档和校验结果。
```
